## 🚑 MIPS 交通救援遊戲 (Traffic Game)
這是一個基於 MIPS 組合語言架構所設計的邏輯棋盤遊戲。玩家將扮演救護車駕駛，在有限的回合內避開老奶奶，成功將所有傷患送達醫院。
### 🎮 遊戲簡介與獲勝條件
* 玩家控制：救護車 🚑  
* 障礙物：老奶奶 👵  
* 目標：將地圖上的 3 位傷患 成功救援並送達「醫院」。  
* 回合限制：遊戲總共只有 32 回合，超過 32 回合未完成即由老奶奶獲勝 (Grandma win)。  
### 🎲 玩法規則  
1. 老奶奶移動：每次移動 1 步。移動方向會受到當前回合數（奇偶性）的限制。若選擇不移動 (輸入 0)，則直接換下一位玩家。   
2. 救護車移動：每次可以移動 2 步。  
3. 判定與碰撞：老奶奶和救護車皆不能超出邊界。老奶奶不能進入醫院，也不能與其他物件重疊。    
### 🗺️ 地圖與座標系統
地圖為一個 8x8 的網格系統，座標使用 8 bits (1 byte) 來表示：  
* X 軸 (低 4 bits)：X2, X1, X0 (由右移 +1 / 左移 -1 控制)  
* Y 軸 (高 4 bits)：Y2, Y1, Y0 (由上移 +16 / 下移 -16 控制)  

地圖元素對應位置  
* 醫院 (紅色)：座標為 17 (十六進位 0x11，即 X=1, Y=1)  
* 邊界 (黃色)：最大邊界值為 119 (二進位 0b01110111，即 X=7, Y=7)  
* 傷患 (橘色)：分散於地圖中，被救援後座標會被標記為 255 (0b11111111) 移出地圖以示完成救援。

### ⚙️ 核心邏輯與流程 (MIPS Implementation)
1. 遊戲初始化 (Initialization)  
從記憶體位址 ($s0) 依序載入所有物件的初始座標與遊戲狀態。包含救護車、老奶奶與三位傷患的位置。  
``` mips
lw $s1, 0($s0)   # ambulance location
lw $s2, 4($s0)   # grandma location
lw $s3, 8($s0)   # injured 1 location
lw $s4, 12($s0)  # injured 2 location
lw $s5, 16($s0)  # injured 3 location
lw $s6, 20($s0)  # num of injured in ambulance
lw $s7, 24($s0)  # num of injured in hospital
lw $t9, 28($s0)  # rounds
```
2. 動態輸入機制 (Syscall Input)  
透過 syscall 5 讓玩家即時從鍵盤輸入移動數值 (如 1 為右移, -1 為左移, 16 為上移, -16 為下移)。利用 $ra 暫存器確保函式能正確返回呼叫點。  
``` mips
move_direction:
    li $v0, 5      # syscall for reading an integer
    syscall
    move $t8, $v0  # 儲存輸入方向至 $t8
    jr $ra         # 返回呼叫此函式的主流程
```
3. 老奶奶回合 (grandma_loop)  
此區塊實作了老奶奶的行動邏輯，並使用大量的相對位址跳躍 (Relative Jump) 來作為輸入錯誤時的迴圈重試機制。  
``` mips
grandma_loop:
    sub $s1, $t0, $zero
    andi $t2, $t0, 1
# 獲取玩家輸入方向
    jal move_direction              # 跳躍至輸入副程式並儲存返回點
    
# 檢查輸入與規則限制
    beq $t8, $zero, ambulance_loop  # 輸入 0：不移動，直接跳至救護車回合
    beq $t2, $zero, 16              # 偶數回合限制：若為偶數，跳至垂直移動檢查
    subi $t1, $t8, 1
    beq $t1, $zero, -20             # 奇數回合限制：不允許橫向右移，錯誤則跳回重新輸入
    subi $t1, $t8, -1
    beq $t1, $zero, -28             # 奇數回合限制：不允許橫向左移，錯誤則跳回
    bne $t2, $zero, 16              # 奇數回合限制：若為奇數，跳過後面的垂直檢查
    subi $t1, $t8, 16
    beq $t1, $zero, -40             # 偶數回合限制：不允許垂直上移，錯誤則跳回
    subi $t1, $t8, -16
    beq $t1, $zero, -48             # 偶數回合限制：不允許垂直下移，錯誤則跳回

# 計算新位置並進行邊界與重疊判定
    add $t0, $t8, $s2               # 計算老奶奶預期的新位置 $t0
    ori $t1, $t0, 0b01110111        # 邊界測試 (最大合法座標為 119)
    subi $t1, $t1, 0b01110111
    bne $t1, $zero, -60             # 若超出 0b01110111 的範圍，代表越界，跳回重新輸入
    beq $t0, $s1, -64               # 碰撞測試：不能踩到救護車，否則跳回
    beq $t0, $s3, -68               # 碰撞測試：不能踩到傷患一
    beq $t0, $s4, -72               # 碰撞測試：不能踩到傷患二
    beq $t0, $s5, -76               # 碰撞測試：不能踩到傷患三
    
# 更新位置與回合交接
    sub $s2, $t0, $zero             # 檢查全數通過，將合法新座標寫入 $s2
    j ambulance_loop                # 結束奶奶回合，換救護車行動

```
邏輯亮點解析：  
* andi $t2, $t0, 1：藉由 AND 運算提取最低位元 (LSB)，藉此判定回合的奇偶數。  
* 負數跳躍 (如 -20, -60)：這是一種用來替代 while 迴圈的底層寫法。當遇到非法移動或越界時，程式計數器 (PC) 會往回跳至 jal move_direction 附近，迫使玩家重新輸入。 
4. 救護車回合 (ambulance_loop)  
救護車具備「連走兩步」、「載入傷患」與「醫院卸客」等複合邏輯。  
``` mips
ambulance_loop:
    addi $t2, $zero, 1               # 設定救護車迴圈計數器
# 獲取方向與邊界測試
    j move_direction                 # 呼叫輸入函式 (此處使用 j 可能依賴前面設計，或配合架構調整)
    add $t0, $s1, $t8                # 計算預期新座標 $t0
    ori $t1, $t9, 0b01110111         # 邊界測試
    subi $t1, $t1, 0b01110111
    bne $t1, $zero, -16              # 若越界則跳回重新輸入
    beq $t0, $s2, -24                # 確保不撞到老奶奶

# 傷患救援判定 (Pick up)
    bne $s6, $zero, 40               # 若車上已經有人 ($s6 != 0)，跳過載人檢查
    bne $t0, $s3, 8
    addi $s6, $s6, 1                 # 成功踩到傷患一：車上人數 +1
    ori $s3, $s3, 0b11111111         # 利用位元運算將傷患一座標標記為 255 (移出地圖)
    bne $t0, $s4, 8
    addi $s6, $s6, 1                 # 踩到傷患二：車上人數 +1
    ori $s4, $s4, 0b11111111
    bne $t0, $s5, 8
    addi $s6, $s6, 1                 # 踩到傷患三：車上人數 +1
    ori $s5, $s5, 0b11111111

    addi $t9, $t9, 1
    j grandma_loop                   # 回到奶奶回合

# 醫院卸載判定 (Drop off) 與勝負判斷
    ori $t1, $t0, 17                 # 檢查是否在醫院座標 (17) 
    subi $t1, $t1, 17
    bne $t1, $zero, -16              # 不是醫院則跳回 (這段可能會因為前面邏輯跳躍而觸發)
    subi $s6, $s6, 1                 # 抵達醫院：卸下傷患，車上人數 -1
    addi $s7, $s7, 1                 # 醫院安置人數 +1
    
    subi $t3, $s7, 3                 # 檢查是否 3 人皆已安置
    beq $t3, $zero, ambulance_win    # 是，觸發救護車獲勝 (Ambulance Win)

# 回合與步數結算
    subi $t2, $t2, 1                 # 扣除步數計數器
    bne $t2, $zero, -100             # 若不為 0，代表還可走另一步，跳回 ambulance_loop 開頭
    addi $t9, $t9, 1                 # 步數用盡，總回合數 +1
    subi $t2, $t9, 32                # 檢查是否達 32 回合上限
    beq $t2, $zero, grandma_win      # 是，觸發老奶奶獲勝 (Grandma Win)
    j grandma_loop                   # 尚未結束，繼續下一個奶奶回合
```
邏輯亮點解析：  
* 傷患清除 (ori X, X, 0b11111111)：當傷患被救上車時，程式巧妙地利用 OR 運算與 255 取聯集，直接將其座標強制改為 255。由於地圖極限只到 119，此舉等同於將該傷患移出地圖，永遠不會再被踩到。  
* 步數計數 ($t2)：利用 $t2 作為步數迴圈控制，當走完一步後 $t2 扣 1，若仍大於零則利用大範圍負數跳躍 (-100) 回到救護車輸入階段，模擬救護車與老奶奶的移動速度差。
